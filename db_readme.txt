=== Описание базы данных ===

- Таблица hosts
Содержит в себе ip адреса серверов, которые когда-либо были найдены при сканировании.

ip_addr -  первичный ключ; столбец, содержащий ip адрес сервера.
last_modified - внешний ключ на таблицу 'events'; соответствует событию в рамках которого последний раз была обновлена информация для данного сервера (см. ниже таблицу 'events').


- Таблица events
Содержит в себе информацию о событиях, связанных с различного рода скаинрованием. Сканирование адресов может производиться разными способами,
поэтому хотелось бы хранить информацию о том, какое сканирование было проведено, когда и каким пользователем (возможно, последнее лишнее).

id - первичный ключ; просто индентификатор.
ts - время появления события.
event_name - внешний ключ на таблицу 'event_types'; имя-индентификатор события.
user - строка с именем пользователя, вносившего изменения в базу данных.
comment - текст комментария к событию (может быть опущено).


- Таблица event_types
Таблица, содержащая статическую информацию о типах возможных событий. Например, разновидностью события может быть получение деталей о Tor узлах.

event_name - первичный ключ; уникальное имя события.
event_description - краткое описание события.


- Таблица open_ports
Таблица, которая содержит информацию обо всех открытых портах всех просканированых серверов.

ip_addr - первичный ключ; внешний ключ на таблицу 'hosts'; ip адрес сервера.
port - первичный ключ; номер открытого порта.
banner - текст, полученный сканнером после отправки запроса к данному серверу к данному порту.


- Таблица tor
Таблица, содержащая различную информацию об узлах сети Tor.

or_addr - первичный ключ; адрес узла.
or_port - первичный ключ; порт узла.
nickname - строка с никнеймом узла.
fingerprint - отпечаток узла.
exit_addresses - внешний ключ на таблицу 'hosts'; адреса, которые узел использует для выхода в открытую сеть.
last_seen - время последнего появления узла в сети.
last_changed_address_or_port - время последнего изменения узлом адреса или порта.
first_seen - время первого появления узла в сети.
running - булевый флаг, показывающий в статус узла.
flags - битовая маска; флаги, выданные узлу авторитетными узлами. Возможные флаги: 'MiddleOnly', 'Authority', 'Stable', 'Valid', 'Guard', 'HSDir', 'V2Dir', 'Fast', 'BadExit', 'Running', 'Exit', 'StaleDesc'.
country - двухбуквенный код страны узла.
verified_host_names - подтвержденные доменные имена.
unverified_host_names - неподтвержденные доменные имена.
contact - контактная информация владельца узла.
